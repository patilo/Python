** 🚀 Domina Python: curso de 0 a backend con python 🚀 **
¡Bienvenido a este repositorio de entrenamiento! Este espacio ha sido diseñado para transformar a estudiantes de primer año en programadores capaces de resolver problemas reales. Aquí no solo aprenderás sintaxis, sino la lógica de ingeniería necesaria para construir software escalable.

📂 Estructura del RoadMap
El contenido está organizado de forma progresiva, permitiéndote avanzar a tu propio ritmo:

**1. 🟢 Nivel Básico (/basico)**
El fundamento de todo gran software. Aquí es donde aprendes a hablar el lenguaje de las máquinas.
Variables y Tipos de Datos: Manejo de strings, enteros, flotantes y booleanos.
Estructuras de Control:
if / elif / else: Toma de decisiones lógica.
match-case: Control de flujo moderno y limpio (Python 3.10+).
Ciclos e Iteraciones:
while: Bucles basados en condiciones.
for: Recorrido eficiente de colecciones y rangos.
Colecciones de Datos:
Listas: Datos ordenados y mutables.
Tuplas: Datos inmutables para seguridad de memoria.
Diccionarios: Estructuras llave-valor (la base del formato JSON).  



**2. 🟡 Funciones y Modularización (/funciones)**
Aprenderás a dejar de escribir código repetitivo y empezar a crear componentes reutilizables.
Definición de Funciones (def): Creación de bloques lógicos.
Parámetros y Retornos: Cómo enviar y recibir información entre funciones.
Scope (Alcance): Entender la diferencia entre variables globales y locales.
Funciones Lambda: Introducción a la programación funcional simplificada.  



**3. 🔴 Métodos y Lógica Avanzada (/metodos)**
Donde la lógica se encuentra con la eficiencia.
Métodos de Strings: Manipulación avanzada de texto.
Métodos de Listas y Diccionarios: Ordenamiento, filtrado y transformación de datos.
Comprensión de Listas (List Comprehension): Código elegante y Pythonic en una sola línea.
Manejo de Errores: Uso de try / except para crear programas robustos que no se detienen ante fallos.  



**4. 🟣 Programación Orientada a Objetos (POO) (/poo)** 
Aquí es donde el código cobra vida. Dejamos de pensar en procesos aislados para empezar a modelar el mundo real mediante Clases y Objetos. Este es el estándar de la industria para construir aplicaciones complejas y escalables.
Clases y Objetos: La plantilla (blueprint) vs. la instancia real.
Atributos y Métodos: Definiendo qué "es" y qué "hace" nuestro objeto (el constructor __init__).
Los 4 Pilares de la POO:
Encapsulamiento: Protegiendo los datos internos (uso de _ y __).
Abstracción: Simplificando la complejidad para el usuario final.
Herencia: Creando jerarquías para reutilizar código de forma inteligente.
Polimorfismo: Permitiendo que diferentes objetos respondan a la misma instrucción de formas distintas.
